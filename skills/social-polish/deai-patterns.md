# 去 AI 味模式与规则

AI 生成的内容有明显的模式特征。本文档列出需要识别和修改的 AI 味模式。

## 核心原则

**AI 味的本质是：**
- 过度追求「正确」和「全面」，失去了人的偏见和锋芒
- 用模板句填充内容，缺乏真实的思考痕迹
- 语气居高临下，像在教育读者而非对话
- 过度使用连接词，像在写学术论文

**去 AI 味的目标是：**
- 像一个有观点的人在说话
- 有情绪、有偏见、有立场
- 口语化但不轻浮
- 信息密度高，不说废话

## 必须删除的模板句

### 空洞开场白

以下句子必须删除或重写：

```
❌ 在当今社会
❌ 在这个快节奏的时代
❌ 随着科技的发展
❌ 随着时代的进步
❌ 在数字化转型的浪潮中
❌ 在信息爆炸的时代
❌ 众所周知
❌ 不可否认的是
❌ 毋庸置疑
❌ 显而易见
❌ 事实上
❌ 实际上（开头使用时）
```

### 空洞过渡句

以下句子必须删除或重写：

```
❌ 那么问题来了
❌ 说到这里你可能会问
❌ 接下来让我们看看
❌ 下面我将详细介绍
❌ 首先我们需要明确
❌ 在深入探讨之前
❌ 要理解这一点，我们首先要
```

### 空洞总结句

以下句子必须删除或重写：

```
❌ 综上所述
❌ 总而言之
❌ 总的来说
❌ 归根结底
❌ 由此可见
❌ 不难看出
```

### 鸡汤式结尾

以下句子必须删除或重写：

```
❌ 让我们共同努力
❌ 相信你一定可以
❌ 只要坚持就一定能成功
❌ 未来可期
❌ 加油
❌ 一起加油吧
❌ 希望这篇文章对你有所帮助
❌ 感谢你的阅读
```

## 必须重写的开头类型

### 定义式开头

```
❌ 原文：写作，是指通过文字表达思想和情感的过程。
✅ 重写：很多程序员觉得自己不需要学写作。毕竟代码才是硬通货，文章算什么？
```

### 背景式开头

```
❌ 原文：随着远程工作的普及，越来越多的人开始思考工作与生活的平衡问题。
✅ 重写：远程工作第三年，我终于明白了一件事：在家工作不是天堂，是另一种地狱。
```

### 铺垫式开头

```
❌ 原文：在讨论这个问题之前，我们需要先了解一些背景知识。
✅ 重写：直接从核心观点开始，背景在需要时穿插。
```

### 反问模板开头

```
❌ 原文：你有没有想过，为什么有些人总是能够成功？
✅ 重写：有个问题困扰了我很久：同样努力，为什么有人成功有人失败？
```

---

## 必须避免的结尾类型

### 总结罗列式

```
❌ 原文：综上所述，我们可以得出以下结论：第一...第二...第三...
✅ 重写：用一个有力的观点或金句收尾，不要罗列。
```

### 鸡汤鼓励式

```
❌ 原文：相信只要你坚持努力，一定能够实现自己的目标。加油！
✅ 重写：给出具体的行动建议，或者抛出引发思考的问题。
```

### 号召式

```
❌ 原文：让我们一起努力，创造更美好的未来！
✅ 重写：说完该说的就结束，不需要喊口号。
```

### 感谢式

```
❌ 原文：感谢你看到这里，希望这篇文章对你有所启发。
✅ 重写：删除，不需要感谢。
```

## 需要控制的元素

### 连接词密度

**规则：每 100 字不超过 2 个以下连接词**

```
然而、但是、不过
此外、另外、而且、同时
因此、所以、于是、故而
首先、其次、再次、最后
一方面...另一方面
不仅...而且
虽然...但是
```

**检查方法：**
1. 全文搜索这些词
2. 计算密度
3. 删除不必要的，或改写句子结构

### 破折号使用

**规则：全文破折号不超过 3 处**

```
❌ 过多使用：这件事——说实话——让我非常震惊——甚至有些不知所措。
✅ 适度使用：这件事让我非常震惊——甚至开始怀疑自己的判断。
```

### 表情符号

**规则：除非风格明确需要，否则不使用表情符号**

```
❌ 这个方法真的很有效！🎉
✅ 这个方法真的很有效。
```

**例外情况：**
- 小红书等平台风格需要时可适度使用
- 但要控制数量，每篇不超过 5 个

### 感叹号

**规则：感叹号要克制使用，每 500 字不超过 2 个**

```
❌ 这太重要了！你一定要记住！千万不要忘记！
✅ 这很重要，值得你认真对待。
```

## 爹味检测与修正

### 什么是爹味

爹味是一种居高临下的说教语气，让读者感觉被教训而非被理解。

### 爹味信号词

```
❌ 你们年轻人要...
❌ 我来告诉你...
❌ 你应该...（高频使用时）
❌ 你必须...
❌ 你一定要...
❌ 记住...（命令语气）
❌ 切记...
❌ 千万不要...
❌ 别傻了
❌ 醒醒吧
```

### 爹味句式

```
❌ 原文：年轻人，你们要学会吃苦。
✅ 重写：我用了三年才明白一个道理：舒适区真的会废掉一个人。

❌ 原文：我告诉你，这个方法才是正确的。
✅ 重写：试了很多方法之后，我发现这个最有效。

❌ 原文：你应该早点明白这个道理。
✅ 重写：这个道理我也是踩了坑才懂的。
```

### 去爹味的核心方法

1. **用「我」代替「你」** - 讲自己的经历，而非教训别人
2. **用经历代替说教** - 「我踩过这个坑」而非「你不要踩这个坑」
3. **承认不确定性** - 「可能」「也许」「在我的经验里」
4. **平等对话语气** - 像朋友聊天，而非老师上课

## 老登味检测

### 什么是老登味

老登味是一种自以为是、显摆资历的语气。

### 老登味信号

```
❌ 作为一个在行业里摸爬滚打了 XX 年的人...
❌ 以我多年的经验来看...
❌ 当年我...（过多回忆往事）
❌ 现在的年轻人不懂...
❌ 我见过太多这样的人了...
```

### 去老登味的方法

1. **减少资历背书** - 让内容本身说话
2. **少用「当年」** - 聚焦当下而非过去
3. **不要炫耀** - 经历是为了说明问题，不是为了显摆
4. **保持谦逊** - 承认自己也在学习

## 自然度检查清单

生成内容后，逐项检查：

### 句式层面

- [ ] 是否有连续 3 个以上相似句式？
- [ ] 是否每段开头都是相似结构？
- [ ] 是否过多使用被动句？
- [ ] 是否有过长的复杂句？（超过 50 字的句子）

### 词汇层面

- [ ] 是否有模板词汇？（见上文清单）
- [ ] 是否用词过于书面？
- [ ] 是否有重复用词？（同一个词出现 3 次以上）
- [ ] 是否有过于抽象的词？（赋能、抓手、打法）

### 语气层面

- [ ] 是否有爹味？
- [ ] 是否有老登味？
- [ ] 是否过于客观中立？（应该有立场）
- [ ] 是否有情绪？（应该有真实情感）

### 结构层面

- [ ] 开头是否直接有料？
- [ ] 是否有过多的铺垫和过渡？
- [ ] 结尾是否干脆利落？
- [ ] 每段是否都有存在价值？

## 快速修正示例

### 示例 1：开头

```
❌ 原文：
在当今这个信息爆炸的时代，写作能力变得越来越重要。无论是在职场中，
还是在日常生活中，良好的文字表达能力都能够帮助我们更好地与他人沟通。

✅ 修正：
程序员最容易忽略的技能是什么？不是某个新框架，不是某门新语言，
而是写作。我花了五年才意识到这一点。
```

### 示例 2：中间段落

```
❌ 原文：
首先，我们需要明确写作的重要性。写作不仅仅是一种表达方式，更是一种思考方式。
通过写作，我们可以更好地整理自己的思路，深化对问题的理解。

✅ 修正：
写作最大的好处不是让别人看到你的想法，而是逼你把想法想清楚。
很多时候我以为自己想明白了，一写才发现全是漏洞。
```

### 示例 3：结尾

```
❌ 原文：
综上所述，写作是一项非常有价值的技能。希望大家能够重视写作的学习，
坚持练习，相信你一定能够成为一个优秀的写作者。让我们一起加油！

✅ 修正：
如果你和几年前的我一样，觉得写作和程序员没什么关系，
那我的建议是：试着把下一个技术方案写成一篇文章。
你会发现，写清楚比写代码难多了。
```

## 使用流程

1. **生成内容后先整体阅读** - 感受是否像人写的
2. **用搜索功能检查模板词** - 快速定位问题
3. **检查开头和结尾** - 这两处 AI 味最重
4. **检查连接词密度** - 每 100 字抽查
5. **朗读一遍** - 听起来自然吗？
6. **最后检查爹味和老登味** - 语气是否平等？
